PROJECT OVERVIEW

About PhishNetra

PhishNetra (meaning "Phishing Eye" in Sanskrit) acts as your digital sentry, constantly scanning domains and links to identify signs of cyber threat. It integrates intelligent heuristics, reputation database calls, and WHOIS domain profile inspections to classify risk instantaneously.

Phishing remains the primary vector for global credential theft. PhishNetra helps users browse confidently by evaluating multiple risk vectors, warning them immediately about invalid certificates, unsafe URLs, and typo-squatted domains before they submit credentials.

CAPABILITIES

Key Security Features

Risk Scoring System

Compiles weight factors to calculate a unified risk score from 0-100 indicating trust status.

URL Heuristics

Scans subdomains, directory structures, and string lengths to identify deceptive URL spoofing.

HTTPS & SSL Check

Instantly reports whether HTTP or HTTPS protocol is utilized and validates cert configuration.

Domain Age Audit

Queries WHOIS data dynamically; identifies newly created sites frequently used in phishing campaigns.

Keyword Detection

Searches for high-frequency terms like 'verify', 'update', 'login', 'free' which attempt social engineering.

Blacklist Integration

Links against active threat feeds and malicious databases to block known phishing hosts instantly.

Warning Interstitials

Halts redirection, loading a high-contrast caution window that alerts users of active risk flags.

Browser Extension

Runs as a background process within Chrome/Firefox with zero friction to user workflow.

EXECUTION PIPELINE

How It Works

01

URL Capture

When you navigate to a website, the Chrome Extension intercepts the link request instantly before page parsing starts, checking threat vector files.

02

Feature Extraction

PhishNetra parses the address for length, subdomains, SSL setup, and executes rapid calls to checking algorithms, returning threat features.

03

Threat Computation

The metrics are calculated through the weight classifier. The engine matches findings against risk guidelines, defining safe, suspicious or dangerous statuses.

04

Real-Time Protection

If the score matches danger criteria, the client is redirected to a custom security blocking page preventing data input or login breaches.
